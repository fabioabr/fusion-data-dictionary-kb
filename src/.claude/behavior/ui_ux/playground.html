Regras de Geração — Estas 20 regras são obrigatórias ao gerar qualquer relatório HTML usando o design system. Extraídas da seção 11 do template oficial.
20 Regras Obrigatórias
Obrigatório
1
Tema escuro é o padrão.
O toggle permite alternar para claro. O HTML inicia sem data-theme (escuro). O tema salvo é restaurado via localStorage.
2
Nunca usar cores fora dos tokens.
Toda cor vem de var(--token). A única exceção é #eaf1f7 no override de light theme para stat-card, intro-text e search-input.
3
Nunca usar frameworks CSS externos.
Sem Bootstrap, Tailwind, Material UI, etc. Apenas CSS customizado com os tokens definidos.
4
Arquivo self-contained.
Um único .html com todo CSS no <style> e todo JS no <script>. Dependências externas: apenas Poppins (fonte) e Remix Icon (ícones) via CDN.
5
Sempre incluir toggle de tema.
O botão fica no .header-right com ícone ri-sun-line (escuro) / ri-moon-line (claro).
6
Sempre responsivo.
Incluir media queries para 1024px e 768px. Grids colapsam, tabs quebram, roadmap vira vertical.
7
Sempre com suporte a impressão.
Incluir @media print: tabs sumem, todos os tab-content ficam visíveis, cards não quebram.
8
Ícones exclusivamente Remix Icon.
Usar classes ri-*. Não usar Font Awesome, Heroicons, Material Icons ou qualquer outra biblioteca.
9
Tipografia exclusivamente Poppins.
Sem outra fonte além de Poppins. Pesos usados: 300, 400, 500, 600, 700.
10
Manter a hierarquia.
Header → Stats Grid → Cards → Sections → Footer. Não inverter a ordem.
11
Placeholders com {{VARIAVEL}}.
Para facilitar substituição programática quando templates são usados em pipelines.
12
Conteúdo em pt-BR como padrão.
Toda label, título e texto em português brasileiro.
13
Sempre disponibilizar 3 idiomas (i18n).
PT-BR (padrão), EN, ES. Usar botões .lang-btn com bandeiras SVG. Atributo data-i18n="key" em todo texto traduzível. Objeto const T = { key: { pt, en, es } }. Função switchLang(lang) com persistência.
14
Mínimo 3 abas obrigatórias.
"Visão Macro" (1a, ativa), "Visão Técnica" (2a) e "Quality Assurance" (3a). Da 4a em diante são opcionais.
15
Visão Macro primeiro.
A aba macro funciona como resumo executivo. Quem lê apenas ela deve entender o cenário em 30 segundos.
16
Visão Técnica com ação.
Cada item na aba técnica deve ter status visual claro e granularidade suficiente para ação imediata.
17
Quality Assurance com transparência.
A aba QA deve exibir score geral, breakdown por critério, pontos fortes e sugestões. Deve documentar por que o score não é 100%.
18
Logo no header.
Sempre usar a logo em base64 via <img class="header-logo">. Suporte a .logo-dark e .logo-light para troca automática por tema.
19
Hover interativo em componentes.
Cards (.card:hover): borda primary + sombra --card-shadow. Stat-cards (.stat-card:hover): borda primary + background --th-bg-hover. Tabelas (thead tr:hover th): background --th-bg-hover na linha inteira. Card-header (.card:hover > .card-header): background --th-bg-hover.
20
Contraste light theme.
No tema claro: body usa --bg: #dde2e8 (cinza), main usa --card-bg: #ffffff (branco), e stat-card/intro-text/search-input usam #eaf1f7 (off-white azulado) para se destacarem.
Exemplo de Implementação i18n (Regra 13)
Troque o idioma nos botões PT / EN / ES no header para ver a tradução em ação neste playground.

Estrutura do objeto de tradução:

const T = { header_title: { pt: 'Design System Playground', en: 'Design System Playground', es: 'Playground del Design System' }, tab_componentes: { pt: 'Componentes', en: 'Components', es: 'Componentes' }, // ... demais chaves }; function switchLang(lang) { currentLang = lang; document.querySelectorAll('[data-i18n]') .forEach(el => { const key = el.getAttribute('data-i18n'); if (T[key] && T[key][lang]) { el.innerHTML = T[key][lang]; } }); // Atualizar botoes ativos document.querySelectorAll('.lang-btn') .forEach(b => b.classList.remove('active')); document.getElementById('lang-' + lang) .classList.add('active'); localStorage.setItem('caefe-lang', lang); }
