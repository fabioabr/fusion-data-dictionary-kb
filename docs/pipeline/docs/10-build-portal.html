10 — Build Portal

Estatísticas consolidadas → docs/index.html
Script Python
Sem LLM
~1s
Objetivo
Esta etapa atualiza a página principal do portal (docs/index.html) com estatísticas consolidadas e module cards — este é o ponto de entrada do GitHub Pages, a primeira página que o usuário vê ao acessar o data dictionary. O portal agrega contagens de tabelas e PVOs por módulo, exibe totais gerais e fornece links diretos para o index.html de cada módulo.
DOR — Definition of Ready
Pré-requisitos para executar esta etapa
  • Todos os indexes de módulo gerados na etapa 09 — docs/fusion-kb/data-dictionary/{MOD}/index.html
  • HTMLs de tabelas existem em docs/fusion-kb/data-dictionary/{MOD}/tables/
  • HTMLs de PVOs existem em docs/fusion-kb/data-dictionary/{MOD}/pvos/ (quando aplicável)
DOD — Definition of Done
Critérios de aceite para esta etapa
  • docs/index.html atualizado com contagens corretas de tabelas e PVOs por módulo
  • Stats grid com totais consolidados (total tabelas, total PVOs, total documentos, módulos ativos)
  • Module cards com pills mostrando contagens de tabelas e PVOs por módulo
  • Links dos module cards apontando para data-dictionary/{MOD}/index.html
  • Conteúdo em 3 idiomas (PT-BR, EN, ES) via data-i18n
Entrada
Arquivos de Entrada

O script conta os arquivos HTML existentes nos diretórios de cada módulo:

Fonte Caminho Descrição
Tables docs/fusion-kb/data-dictionary/*/tables/*.html Contagem de HTMLs de tabelas por módulo
PVOs docs/fusion-kb/data-dictionary/*/pvos/*.html Contagem de HTMLs de PVOs por módulo
Indexes docs/fusion-kb/data-dictionary/*/index.html Existência do index de cada módulo — usado para gerar os links

Arquivo Gerado

docs/index.html

Portal como Ponto de Entrada
Hierarquia de navegação

O portal é a raiz da hierarquia de navegação do data dictionary publicado no GitHub Pages:

docs/index.html ← Portal (esta etapa) └─ docs/fusion-kb/data-dictionary/{MOD}/index.html ← Index do módulo (etapa 09) ├─ tables/{table}.html ← Página da tabela (etapa 07) └─ pvos/{pvo}.html ← Página do PVO (etapa 08)
  • O usuário acessa docs/index.html → vê os módulos disponíveis com contagens
  • Clica em um módulo → navega para data-dictionary/{MOD}/index.html
  • No index do módulo, busca/filtra tabelas ou PVOs → clica para ver o detalhe
  • Na página de detalhe, cross-links levam a outras tabelas/PVOs relacionados
Uso
Comandos de Execução

Execute o script a partir do diretório raiz do Pipeline:

# Atualizar o portal com contagens atuais python build_portal.py # Forçar re-geração completa python build_portal.py --force
Notas Importantes
O portal conta os arquivos .html existentes nos diretórios — se uma etapa anterior não rodou, as contagens estarão zeradas para aquele módulo.
O docs/index.html é o arquivo que o GitHub Pages serve como homepage — qualquer alteração aqui é visível publicamente após push.
Módulos classificados mas sem documentação (ex: Tax, Expenses, Projects) aparecem no portal como cards desabilitados com contagem zero.
Esta é a última etapa do pipeline. Execute somente após completar as etapas 07, 08 e 09 para garantir contagens corretas e links funcionais.
