02 — Extrair PVOs

Excel BICC → JSON por View Object
Script Python
Sem LLM
~3 min por módulo
Objetivo
Extrai Procurement View Objects (PVOs) dos arquivos Excel de mapeamento BICC. Cada PVO representa um View Object no Oracle Fusion que agrupa atributos de uma ou mais tabelas de banco de dados. O script classifica cada PVO por módulo (GL, AP, AR, PO, HCM, SCM, FIN, PRJ, etc.) usando heurísticas sobre o nome do caminho AM, e gera um arquivo JSON por PVO.
DOR — Definition of Ready
Pré-requisitos para executar esta etapa
  • Arquivos Excel BICC existem em src/raw-files/ (FSCM e/ou HCM)
  • Python 3.12+ instalado com a biblioteca openpyxl
  • Script extract_pvos.py disponível no diretório da etapa 02
DOD — Definition of Done
Critérios de aceite para esta etapa
  • Um arquivo .json por PVO criado em docs/{MOD}/pvos/
  • JSON contém meta (pvo_name, full_path, module_code, description), stats, tables, attributes e by_table
  • Classificação de módulo completa — 17 módulos mapeados, <1% classificado como OTHER
Entrada
Arquivos de Entrada
Arquivo Caminho Descrição
FSCM src/raw-files/Rel13_25A_BICC_FSCM_Database_Mapping_with_ViewObjects.xlsx Database Mapping de módulos financeiros: GL, AP, AR, PO e outros
HCM src/raw-files/Rel13_25A_BICC_HCM_Database_Mapping_with_ViewObjects.xlsx Database Mapping de módulos de recursos humanos: HCM e submodules
Heurísticas MODULE_HINTS + AM_MODULE_MAP (embutidos no script) Dicionários de classificação de módulo por nome AM e palavras-chave no path
Saída
Arquivos Gerados

Padrão de saída por PVO:

src/kb/oracle-fusion-data-dictionary/docs/{MOD}/pvos/{pvo_name}.json

Exemplo completo — BalancePVO.json (JSON gerado):

{ "meta": { "pvo_name": "BalancePVO", "full_path": "FscmTopModelAM.FinGlBiccExtractAM.BalancePVO", "module_path": "FscmTopModelAM.FinGlBiccExtractAM", "module_code": "GL", "type": "pvo", "description": "(preenchido na etapa 04)" }, "stats": { "total_attributes": 28, "tables_count": 1, "bicc_enabled": 20, "bicc_pct": 71, "pk_count": 0 }, "tables": ["GL_BALANCES"], "attributes": [ { "name": "LedgerId", "table": "GL_BALANCES", "column": "LEDGER_ID", "bicc": true, "pk": false, "incremental": false } ], "by_table": { "GL_BALANCES": ["LedgerId", "..."] } }
Como Funciona
Passo a passo interno do script
  1. Lê todas as planilhas dos arquivos Excel FSCM e HCM usando openpyxl
  2. Agrupa as linhas por caminho do View Object (coluna A)
  3. Para cada VO, extrai atributos: nome, tabela, coluna, BICC habilitado, PK, incremental
  4. Classifica o módulo usando heurística de 3 camadas: (a) match exato do nome AM em AM_MODULE_MAP, (b) match por keyword em MODULE_HINTS sobre o nome AM, (c) match por keyword no path completo
  5. Constrói o JSON com meta, stats, tables, attributes e agrupamento by_table
  6. Grava o arquivo .json no diretório pvos/ específico do módulo
Classificação de Módulos
17 módulos identificados — 4.239 PVOs classificados

A classificação utiliza heurísticas sobre o caminho AM (Application Module) do View Object. O algoritmo de 3 camadas garante cobertura de >99% dos PVOs, com menos de 1% classificado como OTHER.

Uso
Comandos de Execução

Execute o script a partir do diretório da etapa 02:

# Extrair PVOs de um módulo específico cd Pipeline/02-extract-pvos/ python extract_pvos.py --module GL # Extrair PVOs de HCM python extract_pvos.py --module HCM # Extrair todos os PVOs de todos os módulos python extract_pvos.py --all
Notas Importantes
O campo description no JSON é gerado automaticamente (genérico) nesta etapa — será enriquecido por LLM na etapa 04.
Os arquivos Excel contêm aproximadamente 3.400 View Objects no total, distribuídos entre FSCM e HCM.
A classificação de módulo utiliza heurísticas sobre o caminho AM. O dicionário AM_MODULE_MAP cobre matches exatos; MODULE_HINTS cobre matches por keyword. Menos de 1% dos PVOs ficam como OTHER.
