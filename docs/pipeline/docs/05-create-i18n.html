05 — Criar Arquivos i18n

.md / .json → .i18n.json
Script Python
Sem LLM
~30s por módulo
Objetivo
Esta etapa extrai conteúdo traduzível de arquivos .md (tabelas) e .json (PVOs) e gera arquivos .i18n.json estruturados com slots para PT-BR, EN e ES. Os textos em pt-BR são preenchidos automaticamente a partir do conteúdo fonte; os campos EN e ES ficam como null para serem preenchidos na etapa 06 (Traduzir). Nenhum LLM é utilizado — é pura extração e estruturação de dados.
DOR — Definition of Ready
Pré-requisitos para executar esta etapa
  • Arquivos .md enriquecidos pela etapa 03 existem em src/kb/.../docs/{MOD}/tables/
  • Arquivos .json de PVOs com descrições da etapa 04 existem em src/kb/.../docs/{MOD}/pvos/
  • Python 3.12+ instalado (sem dependências externas)
DOD — Definition of Done
Critérios de aceite para esta etapa
  • Um arquivo .i18n.json ao lado de cada .md e .json fonte
  • Formato: {source, source_hash, translated_at, items: [{id, br, en:null, es:null}]}
  • Campo br preenchido com texto extraído do fonte; en e es como null
  • Hash MD5 do arquivo fonte registrado em source_hash para controle de cache
Entrada
Arquivos de Entrada
Arquivo Caminho Descrição
Tabelas .md src/kb/.../docs/{MOD}/tables/*.md Tabelas enriquecidas com descrições (visão geral, colunas, relacionamentos)
PVOs .json src/kb/.../docs/{MOD}/pvos/*.json PVOs com campo meta.description preenchido em pt-BR
Saída
Arquivos Gerados

Um arquivo .i18n.json é criado ao lado de cada fonte:

src/kb/.../docs/GL/tables/gl_balances.md → gl_balances.i18n.json src/kb/.../docs/HCM/pvos/PersonPVO.json → PersonPVO.i18n.json

Como Funciona
Passo a passo interno

Para tabelas (.md):

  1. Lê o arquivo .md e extrai a seção "Visão Geral" como item overview
  2. Extrai as descrições de cada coluna da tabela de colunas como itens col_{nome}
  3. Extrai descrições de relacionamentos (pai/filha) como itens rel_{tabela}
  4. Calcula o hash MD5 do conteúdo do arquivo fonte para controle de cache
  5. Gera o .i18n.json com todos os itens e campos en/es como null

Para PVOs (.json):

  1. Lê o arquivo .json e extrai o campo meta.description como item description
  2. Calcula o hash MD5 do conteúdo do arquivo fonte
  3. Gera o .i18n.json com o item de descrição e campos en/es como null
Dois Scripts
Scripts de Extração
Script Entrada Saída
extract_table_i18n.py Arquivos .md de tabelas Arquivos .i18n.json com overview + colunas + relacionamentos
extract_pvo_i18n.py Arquivos .json de PVOs Arquivos .i18n.json com descrição do PVO
Cache por Hash
Estratégia de Cache com MD5

Cada .i18n.json armazena o hash MD5 do arquivo fonte no campo source_hash. Ao re-executar o script, se o hash do fonte atual for igual ao registrado, o arquivo .i18n.json é pulado — evitando reprocessamento desnecessário e preservando traduções já realizadas na etapa 06.

Use a flag --force para forçar a regeneração de todos os .i18n.json, ignorando o cache de hash. Isso é útil quando a estrutura do .i18n.json muda entre versões do script.
Uso
Comandos de Execução

Execute os scripts a partir do diretório da etapa 05:

# Extrair i18n de tabelas de um módulo cd Pipeline/05-create-i18n/ python extract_table_i18n.py --module GL # Extrair i18n de PVOs de um módulo python extract_pvo_i18n.py --module HCM # Processar todos os módulos python extract_table_i18n.py --all python extract_pvo_i18n.py --all # Forçar regeneração (ignorar cache de hash) python extract_table_i18n.py --module GL --force
Notas Importantes
Esta etapa NÃO traduz nenhum conteúdo — apenas extrai os textos em pt-BR e cria a estrutura para tradução. A tradução acontece na etapa 06.
Se o arquivo .i18n.json já existe e o hash do fonte não mudou, o script pula a regeneração para preservar traduções já feitas.
Usar --force em um arquivo que já foi traduzido (etapa 06) vai apagar as traduções EN/ES, resetando os campos para null. Use com cuidado.
