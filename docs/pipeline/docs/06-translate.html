06 — Traduzir EN/ES

i18n.json (pt-BR) → i18n.json (pt-BR + EN + ES)
Ollama ou Opus
~30min Ollama / ~3min Opus
Objetivo
Esta etapa preenche os campos en (inglês) e es (espanhol) nos arquivos .i18n.json gerados na etapa 05. Duas estratégias estão disponíveis: Ollama local (gratuito, mais lento) ou Opus paralelo (via API, mais rápido). Termos técnicos do domínio Oracle Fusion são mantidos em inglês em todas as traduções para preservar a consistência do glossário técnico.
DOR — Definition of Ready
Pré-requisitos para executar esta etapa
  • Arquivos .i18n.json da etapa 05 existem com campo br preenchido e en/es como null
  • Ollama rodando localmente com modelo translategemma:4b OU API key Anthropic configurada
  • Python 3.12+ instalado com requests (para Ollama) ou Claude Code (para Opus)
DOD — Definition of Done
Critérios de aceite para esta etapa
  • Todos os campos en e es preenchidos em todos os .i18n.json
  • Termos técnicos mantidos em inglês (workflow, business unit, subledger, posting, etc.)
  • Campo translated_at atualizado com timestamp da tradução
  • Nenhum campo en ou es permanece como null
Entrada
Arquivos de Entrada
Arquivo Caminho Descrição
i18n src/kb/.../docs/{MOD}/tables/*.i18n.json Arquivos i18n de tabelas com br preenchido, en/es null
i18n src/kb/.../docs/{MOD}/pvos/*.i18n.json Arquivos i18n de PVOs com br preenchido, en/es null
Saída
Arquivos Atualizados

Os mesmos .i18n.json são atualizados in-place com en e es preenchidos:

{ "source": "gl_balances.md", "source_hash": "a3f2b8c1d4e5f6a7b8c9d0e1f2a3b4c5", "translated_at": "2026-03-25T14:32:00Z", "items": [ { "id": "overview", "br": "Armazena os saldos contábeis por período, ledger e combinação de contas.", "en": "Stores accounting balances by period, ledger, and account combination.", "es": "Almacena los saldos contables por período, ledger y combinación de cuentas." }, { "id": "col_ledger_id", "br": "Referência ao ledger (razão contábil) ao qual o saldo pertence.", "en": "Reference to the ledger to which the balance belongs.", "es": "Referencia al ledger al cual pertenece el saldo." } ] }
Duas Estratégias
Ollama Local vs. Opus Paralelo
Aspecto Ollama Local Opus Paralelo
Script translate_json.py --batch Claude Code + Agent tool
Modelo translategemma:4b Claude Opus
Custo Gratuito API tokens
Velocidade ~30 min (2.391 arquivos) ~3 min (4 agentes paralelos)
Qualidade Boa Excelente
Como Funciona — Ollama
Tradução via Ollama Local
  1. Coleta todos os .i18n.json que ainda possuem en ou es como null
  2. Agrupa os textos pendentes em batches de 25 itens numerados
  3. Envia cada batch para o Ollama via REST API (http://localhost:11434/api/generate) com prompt: "Translate each numbered text to {lang}. Keep technical terms in English."
  4. Recebe a resposta com textos numerados e faz o parse de volta para os itens originais
  5. Atualiza os campos en e es nos .i18n.json e registra o translated_at
Como Funciona — Opus
Tradução via Opus Paralelo
  1. Consolida todos os itens pendentes (com en/es null) em um único JSON
  2. Divide os itens em 4 chunks balanceados
  3. Lança 4 agentes Opus em paralelo, cada um traduzindo seu chunk para EN e ES simultaneamente
  4. Cada agente retorna um JSON com {id: {en: "...", es: "..."}}
  5. Os resultados são consolidados e escritos de volta nos .i18n.json individuais
Termos Técnicos (manter em inglês)
Glossário de Termos Preservados

Os seguintes termos técnicos são mantidos em inglês em todas as traduções (EN e ES), para preservar a consistência com a documentação Oracle e o glossário corporativo:

workflow business unit subledger posting BICC payroll HCM date-effective ledger journal batch invoice purchase order View Object Application Module PVO flex field lookup setup staging interface header line distribution

Uso
Comandos de Execução

Via Ollama (recomendado para batch):

# Traduzir todos os i18n pendentes de um módulo cd Pipeline/06-translate/ python translate_json.py --batch --module GL # Traduzir todos os módulos python translate_json.py --batch --all # Traduzir apenas EN (pular ES) python translate_json.py --batch --module GL --lang en

Via Opus (mais rápido, usa API):

# Via Claude Code — traduzir com agentes paralelos # (dentro do Claude Code, usar Agent tool) # Prompt para o agente: # "Leia todos os .i18n.json pendentes em src/kb/.../docs/GL/, # traduza os campos br para en e es, # mantendo termos técnicos em inglês, # e atualize os arquivos."
Notas Importantes
Traduções já preenchidas (en/es não-null) são puladas por padrão. Use --force para retraduzir tudo.
O Ollama processa em batches de 25 textos numerados para maximizar throughput sem perder a correlação entrada/saída.
Se o Ollama retornar menos linhas do que o batch enviado, o script identifica os itens faltantes e os reenvia em um batch de retry.
A estratégia Opus paralela é ~10x mais rápida que Ollama, mas consome tokens da API. Recomendada para runs iniciais; Ollama para re-traduções pontuais.
