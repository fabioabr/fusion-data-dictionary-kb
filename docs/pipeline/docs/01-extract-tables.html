01 — Extrair Tabelas

Excel BICC → Markdown Skeleton
Script Python
Sem LLM
~5 min por módulo
Objetivo
Esta etapa lê os arquivos Excel de Database Mapping do Oracle BICC e gera um arquivo .md por tabela, contendo metadados estruturados: front matter YAML, lista de colunas com tipos e flags (PK/FK), e seções vazias para relacionamentos e SQL. Nenhuma descrição é gerada nesta fase — apenas a estrutura esquelética. As descrições são preenchidas na etapa 03 (Descrever Tabelas) via LLM.
DOR — Definition of Ready
Pré-requisitos para executar esta etapa
  • Arquivos Excel BICC existem em src/raw-files/ (FSCM e/ou HCM)
  • Arquivo table-mappings.txt existe com a classificação de módulos por tabela
  • Python 3.12+ instalado com a biblioteca openpyxl
DOD — Definition of Done
Critérios de aceite para esta etapa
  • Um arquivo .md por tabela criado em src/kb/.../docs/{MOD}/tables/
  • Front matter com campos: id, doc_type, system, module, title, status, confidentiality
  • Tabela de colunas com: #, Column, Type, Nullable, Category, Description (vazio), FK, Confidence
  • Seções vazias para Relationships, SQL e Filters criadas como placeholder
Entrada
Arquivos de Entrada
Arquivo Caminho Descrição

Como Funciona
Passo a passo interno do script
  1. Lê as planilhas do Excel (sheets de BI View Object columns) usando openpyxl
  2. Agrupa as linhas por nome de tabela (coluna Database Table)
  3. Consulta a classificação de módulo no arquivo table-mappings.txt
  4. Para cada tabela, gera o front matter YAML com metadados do módulo (id, doc_type, system, module, title)
  5. Cria a tabela de colunas a partir dos dados do Excel (nome, tipo, nullable, flag PK)
  6. Detecta colunas FK por convenção de nomes (*_ID referenciando outras tabelas conhecidas)
  7. Cria seções vazias de relacionamentos (pai/filha), SQL e filtros como placeholders
  8. Grava o arquivo .md no diretório do módulo (pula se já existe, exceto com --force)
Uso
Comandos de Execução

Execute o script a partir do diretório da etapa 01:

# Extrair tabelas de um módulo específico cd Pipeline/01-extract-tables/ python extract_tables.py --module GL # Extrair tabelas de HCM python extract_tables.py --module HCM # Extrair todas as tabelas de todos os módulos python extract_tables.py --all # Forçar re-extração (sobrescreve .md existentes) python extract_tables.py --module GL --force
Notas Importantes
Esta etapa NÃO gera descrições — apenas a estrutura esquelética (colunas, tipos, PKs/FKs). As descrições são preenchidas na etapa 03 via LLM.
Arquivos .md já existentes NÃO são sobrescritos por padrão. Use a flag --force para forçar a re-extração.
O arquivo table-mappings.txt define qual tabela pertence a qual módulo. Tabelas não mapeadas são ignoradas ou classificadas como "Other".
Os arquivos Excel contêm ~3.400 caminhos PVO, mas esta etapa extrai apenas os metadados de TABLE (não PVOs). A extração de PVOs ocorre na etapa 02.
