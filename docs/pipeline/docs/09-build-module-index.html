09 — Build Index por Módulo

Metadados → index.html com tabs e busca
Script Python
Sem LLM
~1s por módulo
Objetivo
Esta etapa gera um index.html por módulo que serve como landing page do data dictionary daquele módulo. O index consolida estatísticas (total de tabelas, PVOs, colunas), oferece abas para Tabelas e PVOs (quando existem), campo de busca/filtro em tempo real e links diretos para cada documento HTML individual. É o ponto de entrada ao navegar dentro de um módulo específico.
DOR — Definition of Ready
Pré-requisitos para executar esta etapa
  • Arquivos .md de tabelas existem com metadados no front matter
  • Arquivos .json de PVOs existem com metadados (quando o módulo possui PVOs)
  • HTMLs de tabelas gerados na etapa 07 — docs/fusion-kb/data-dictionary/{MOD}/tables/*.html
  • HTMLs de PVOs gerados na etapa 08 — docs/fusion-kb/data-dictionary/{MOD}/pvos/*.html (quando aplicável)
DOD — Definition of Done
Critérios de aceite para esta etapa
  • Um index.html por módulo em docs/fusion-kb/data-dictionary/{MOD}/
  • Abas para Tables e PVOs (PVOs tab exibida apenas se o módulo tiver PVOs)
  • Campo de busca funcional que filtra por nome, descrição, número de colunas e status
  • Stats grid com contagens de tabelas, PVOs e total de colunas
  • Conteúdo em 3 idiomas (PT-BR, EN, ES) via data-i18n
Entrada
Arquivos de Entrada
Arquivo Caminho Descrição
.md src/kb/oracle-fusion-data-dictionary/docs/{MOD}/tables/*.md Metadados do front matter: title, module, status, contagem de colunas
.json src/kb/oracle-fusion-data-dictionary/docs/{MOD}/pvos/*.json Metadados de PVOs: nome, descrição, contagem de atributos e tabelas-base
Mapeamento src/raw-files/table-mappings.txt Mapeamento tabela → módulo para resolução de nomes completos de módulo
Saída
Arquivos Gerados

Padrão de saída por módulo:

docs/fusion-kb/data-dictionary/{MOD}/index.html

Estrutura do index.html gerado:

Seção Conteúdo
Header Nome do módulo com ícone, breadcrumb (Portal → Módulo), seletor de idioma, toggle tema
Stats Grid KPIs consolidados: total de tabelas, total de PVOs, total de colunas, status (completed/pending)
Tab Tables Tabela pesquisável com nome, descrição, colunas, status — cada linha é link para o HTML da tabela
Tab PVOs Tabela pesquisável com nome, descrição, atributos, tabelas-base — cada linha é link para o HTML do PVO
Tabs Condicionais
Lógica de exibição de abas

As abas são exibidas condicionalmente com base na existência de PVOs no módulo:

  • Módulo com PVOs (GL, AP, AR, PO, HCM) — exibe sistema de abas Tables + PVOs com badge de contagem em cada aba
  • Módulo sem PVOs — exibe layout plano (flat) apenas com a tabela de tabelas, sem sistema de abas
  • Caso HCM — inicialmente sem PVOs (layout flat), após inclusão dos 841 PVOs passou a usar sistema de abas
Uso
Comandos de Execução

Execute o script a partir do diretório raiz do Pipeline:

# Gerar index de todos os módulos python build_module_index.py # Gerar index de um módulo específico python build_module_index.py --module AR # Forçar re-geração python build_module_index.py --module GL --force
Notas Importantes
O index lê os metadados diretamente dos arquivos .md e .json — não depende dos HTMLs gerados, mas os links apontam para eles.
O campo de busca filtra em tempo real por nome da tabela/PVO, descrição e status — sem necessidade de pressionar Enter.
A contagem de colunas é extraída do .md parseando a tabela Markdown de colunas — não do Excel original.
Se HTMLs de tabelas ou PVOs ainda não foram gerados (etapas 07/08), os links no index resultarão em 404. Execute as etapas na ordem correta.
